LINUX FOR DEVOPS

What is Linux?

Linux is an operating system similar to Windows, but
it is mostly used in servers, cloud platforms and DevOps.

In real companies most servers run Linux because:

• Free and open source
• Lightweight
• Secure
• Supports automation
• Easy remote access
• Better performance
• Supports DevOps tools

Examples of Linux Distributions:

Ubuntu
CentOS
RedHat (RHEL)
Debian
Amazon Linux
Kali Linux

----------------------------------------------------

Where Linux is used?

• AWS EC2 servers
• Web servers
• Database servers
• Docker containers
• Kubernetes clusters
• Cloud environments
• Networking systems

LINUX ARCHITECTURE

              User
                ↓
             Shell
                ↓
             Kernel
                ↓
Hardware (CPU RAM Disk)

User

The person giving commands.

Example:

----------------------------------------------------

Shell

Acts as middle layer between user and Linux system.

It accepts commands and passes them to kernel.

Examples:

bash
zsh
sh

Example:

ls

Shell understands command and sends request.

----------------------------------------------------

Kernel

Kernel is the heart of Linux.

Responsibilities:

• Memory management
• Process management
• Device management
• File management
• CPU scheduling

----------------------------------------------------

Hardware

Physical components:

• CPU
• RAM
• Hard Disk
• Network card

PERMISSIONS

Linux permissions:

Read = r = 4
Write = w = 2
Execute = x =1

Example:

-rwxr-xr-x

Meaning:

Owner → rwx

Group → r-x

Others → r-x

Permission values:

777

Full access

755

Owner full access
Others read and execute

644

Owner read/write
Others read only

Commands:

chmod 755 file.sh

chmod +x script.sh

chown user file.txt

chgrp developers file.txt

REAL DEVOPS SCENARIO

Problem:

Website is down.

Steps:

1. SSH into server

ssh ubuntu@IP

2. Check process

ps -ef

3. Check logs

cd /var/log

tail -f application.log

4. Restart service

sudo systemctl restart nginx

5. Verify status

sudo systemctl status nginx
